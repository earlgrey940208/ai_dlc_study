## Questions

### Question 1: Story Breakdown 방식
스토리를 어떤 방식으로 분류하시겠습니까?

A) User Journey 기반 - 사용자 흐름 순서대로 (로그인 → 메뉴 조회 → 장바구니 → 주문)
B) Feature 기반 - 시스템 기능 단위로 (인증, 메뉴, 장바구니, 주문, 모니터링)
C) Persona 기반 - 사용자 유형별로 (고객 스토리 / 관리자 스토리)
D) AI 추천에 맡기기

[Answer]:B

---

### Question 2: Acceptance Criteria 상세도
각 스토리의 Acceptance Criteria를 얼마나 상세하게 작성할까요?

A) 간결하게 - 핵심 조건만 (Given/When/Then 1~2개)
B) 표준 - 정상/예외 케이스 포함 (Given/When/Then 3~5개)
C) 상세하게 - 엣지 케이스, UI 동작까지 포함

[Answer]:A

---

### Question 3: Story 우선순위 표기
스토리에 우선순위를 표기할까요?

A) 예 - Must/Should/Could (MoSCoW 방식)
B) 예 - High/Medium/Low
C) 아니오 - 우선순위 없이 기능별로만 나열

[Answer]:B

---

### Question 4: 관리자 페르소나 세분화
관리자 역할을 세분화할까요?

A) 단일 관리자 - 매장 사장님이 모든 관리 수행
B) 복수 관리자 - 사장님 + 직원(홀 매니저) 구분
C) AI 추천에 맡기기

[Answer]:A
